# 违章记录列表移除处理状态设计

## 背景

违章记录模块只承担正式违章档案的查询与展示，不包含缴费、撤销、结案等后续处理流程。当前列表展示 `pending/handled/cancelled` 状态，但系统没有更新这些状态的业务入口，导致所有记录长期显示为“待处理”。

## 目标

- 从审核员端违章记录列表移除“状态”筛选。
- 从审核员端违章记录列表移除“状态”列。
- 保留车牌号、违章类型、地点、发生时间、罚款和扣分等正式违章信息。

## 非目标

- 不删除数据库 `violations.status` 字段。
- 不修改后端违章查询 API 及其兼容性。
- 不修改管理员端或市民端的其他违章页面。
- 不新增缴费、撤销或处理完成流程。

## 前端改动

修改 `smart-traffic-frontend/src/views/review/ViolationList.vue`：

1. 删除筛选表单中的状态下拉框。
2. 删除表格中的状态列。
3. 删除页面内不再使用的 `statusMap`。
4. 从筛选对象和重置逻辑中删除 `status`。

通用查询构造函数 `buildViolationQuery` 继续支持 `status`，以免影响其他调用方及后端 API 兼容性。

## 验证

- 前端测试通过。
- 前端构建通过。
- `/review/violations` 不再出现状态筛选和状态列。
- 查询、重置和分页仍正常工作。
