## 8. 前端页面设计

### 8.1 市民端

- 登录/注册。
- 随手拍上传。
- 举报进度列表。
- 本人车辆绑定。
- 本人违章查询。

### 8.2 审核工作台

核心页面：案件卡片流。

能力：

- 实时弹出新案件卡片。
- 支持按状态、地点、时间、来源筛选。
- 支持查看原图、标注图和 AI 说明。
- 支持通过、驳回、补录车牌、修改违章类型。
- 支持查看 AI 复核链路：YOLOv8 目标检测结果、OCR 结果、规则判定结果、LLM 初审结果。

实时方式：

- 首选 WebSocket/SSE 推送 `case.created`。
- 若连接失败，前端每 10 秒轮询 `/cases?status=pending_human_review`。

### 8.3 统计分析台

- 总案件数、通过率、驳回率、通知成功率。
- 按地点排行。
- 按时间趋势。
- 按违章类型占比。
- LLM 周报/月报生成入口。

### 8.4 系统管理

- 用户和角色管理。
- 摄像头设备和 API Key 管理。
- 违章规则维护。
- 短信模板和供应商配置。
- 操作日志查看。

## 9. 异步任务设计

| 任务 | 触发时机 | 结果 |
| --- | --- | --- |
| `trigger_candidate_event_task` | 摄像头视频流接入后 | 从高频帧中筛选疑似违章候选事件 |
| `detect_objects_task` | 候选事件生成后 | 生成 YOLOv8 目标检测结果 |
| `ocr_plate_task` | YOLOv8 识别到车牌框后 | 生成车牌候选 |
| `evaluate_violation_rule_task` | YOLOv8 和 OCR 完成后 | 生成违章类型候选、规则匹配结果和证据完整度 |
| `ai_review_task` | 规则判定完成后 | 生成 AI 初审结果 |
| `send_sms_task` | 违章归档后 | 发送短信通知 |
| `generate_report_task` | 用户发起报告生成 | 生成 LLM 分析报告 |

任务失败处理：

- AI 任务失败：案件进入 `pending_human_review`，卡片显示“AI 初审失败，需人工判断”。
- 短信失败：记录失败原因并重试，超过次数后状态为 `failed`。
- 文件存储失败：接入接口返回失败，不创建案件。

## 10. 安全与审计

- JWT 登录鉴权。
- RBAC 控制菜单和接口权限。
- 摄像头接口使用 API Key + 签名 + 时间戳防重放。
- 图片限制格式、大小和 MIME 类型。
- 敏感接口记录 `audit_logs`。
- AI 原始结果不可人工覆盖，只能追加人工审核结论。
- 短信内容使用模板生成，避免任意文本发送。

## 11. MVP 实施顺序

1. 搭建 FastAPI + Vue3 基础项目和登录权限。
2. 完成图片上传、文件存储、案件表和案件列表。
3. 接入 YOLOv8，生成目标检测结果和标注图。
4. 完成违章规则判定服务，基于视觉要素和业务规则生成违章候选。
5. 完成文本 LLM 初审和多模态复核分流。
6. 完成审核工作台案件卡片、通过、驳回。
7. 完成违章归档和短信通知适配器。
8. 完成统计接口和 LLM 分析报告。
9. 补充权限、日志、异常处理和演示数据。

## 12. 验收标准

- 市民上传和摄像头上传均能进入统一案件流水线。
- 摄像头 20fps 高频视频流必须先筛选候选事件，普通非候选帧不进入 LLM 或多模态模型。
- YOLOv8 只输出目标检测结果，系统不使用目标检测置信度直接判断违章。
- 规则匹配且证据完整的案件能生成 AI 初审意见并进入人工终审。
- 只有疑似违章但证据缺失、空间关系不清或图片复杂的候选事件才调用多模态模型复核。
- 明显不满足违章规则的图片或候选事件不生成案件卡片。
- 审核工作台能弹出或刷新显示案件卡片。
- 人工通过后生成正式违章记录。
- 归档后能调用真实短信接口并记录通知结果。
- 车管/交管工作人员能查看每条违章信息和统计数据。
- 系统能基于一段时间统计数据生成 LLM 分析报告。
