# 🚦 基于 AI 的交通违章智能管理平台

> **YOLO‑V8 + LLM 大模型驱动的交通违章智能管理平台**

本项目面向交通管理部门，融合 YOLO‑V8 视觉检测与 LLM 大模型初审能力。系统通过「AI 自动检测 → LLM 智能初审 → 人工复核确认」三层架构，实现对三类图片来源（外部摄像头推送、管理员上传、市民随手拍）的违章图片进行智能识别与审核，最终关联车辆‑车主信息，汇聚至数据可视化大屏。

---

## 📖 项目概述

基于 AI 的交通违章智能管理平台支持完整的违章处理业务闭环：三类图片来源提交后，YOLO‑V8 进行交通目标及疑似违章行为特征检测，LLM 大模型智能体进行 AI 初审并输出标准化审核建议，审核管理员对存疑记录进行人工复核，通过后生成正式违章记录、关联车辆‑车主信息、发放随手拍奖励，最终数据汇总至可视化大屏并自动生成分析报告。

项目采用 **Vue 3** + **FastAPI** + **YOLOv8** + **LLM 大模型** 前后端分离架构。

系统支持三种图片接入方式：

| 来源 | 说明 |
|------|------|
| **外部路口电子摄像头** | 外部抓拍程序调用后端鉴权 API 接口，自动推送抓拍图片、车速、抓拍地点、时间 |
| **审核管理员手动上传** | 审核管理员在后台管理页面手动上传图片（用于测试、补充证据等场景） |

## ✨ 功能特性（六大模块）

### 1. 用户及权限管理模块

- 三种角色用户均可完成注册、登录、修改个人信息
- **RBAC 权限模型**：运维人员分配权限，侧边菜单栏、操作按钮根据登录角色动态渲染
- 系统记录关键操作日志，支持发布系统公告
- 对外部摄像头调用的 API 接口设置**独立密钥鉴权**，阻止非法程序访问
- 做好举报人隐私保护，防止用户刷取奖励（防重复举报机制）

### 2. 车辆与驾驶员（车主）管理模块

- 审核管理员手动录入车辆及驾驶员绑定手机号信息（模拟外部系统数据导入）
- 将违章记录与车主手机信息进行关联，实现违章处罚落地
- 为数据大屏、LLM 智能体提供基础统计数据
- 车主收到处罚通知：包含当前分数、被扣分数、处罚原因（短信通知）

### 3. 违章管理模块（核心业务模块 ★）

#### 3.1 图片接入
- 接收外部摄像头 API 推送的图片（车速、地点、时间）
- 审核管理员后台手动上传图片
- 市民随手拍举报上传（填写违章地点、时间描述）

#### 3.2 YOLO‑V8 图像识别
- 对所有上传图片进行交通目标及疑似违章行为特征检测
- 识别违章类型、定位车牌区域，生成标注框
- 输出识别置信度

#### 3.3 LLM 大模型 AI 初审
- 综合 YOLO 识别结果、置信度、车速、路段限速信息
- 判定每条记录为「建议直接通过」或「存疑人工复核」，生成文字判定理由
- 输出标准化审核参考意见，辅助审核管理员快速筛选待处理记录

#### 3.4 审核工作流
- 审核管理员复核违章记录：审核通过 / 驳回操作，可填写处理意见
- 支持**批量审核**
- 支持**导出 Excel 违章案卷**
- 所有审核完成的违章记录绑定车辆‑车主信息

#### 3.5 随手拍举报子功能
- 市民填写违章地点、时间描述，上传照片提交举报
- 系统经过 YOLO 识别 → LLM 初审 → 审核管理员核验
- 举报核实有效 → 生成违章记录 + 发放对应奖励
- 举报无效 → 驳回（可查看驳回原因）
- 运维人员可配置单笔有效举报的奖励标准
- 市民可查看举报进度与奖励明细

#### 3.6 查询功能
- 审核管理员可查看所有车辆违规信息
- 支持按指定用户、指定时段、指定地点查看违规记录

### 4. LLM 大模型智能体拓展模块

| 功能 | 说明 | 优先级 |
|------|------|:------:|
| **AI 违章统一初审** | 对三类来源的全部违章记录统一完成智能初审；结合 YOLO 识别结果 + 车速 + 路段信息，输出标准化审核参考意见，标记「建议直接通过」或「存疑人工复核」 | P0 |
| **AI 问答工具** | 审核管理员可询问具体判定原因及相关法规判定规则（与 AI 初审结论联动） | P0 |
| **自动生成分析报告** | 自动生成周/月度交管文字分析报告，附带统计图表，支持 PDF 导出 | P1 |
| **交通管控整治方案** | 针对违章高发路段，自动生成交通管控整治方案建议 | P2 |

### 5. 数据可视化与综合查询模块

- **数据可视化大屏**：展示核心统计指标、违章趋势折线图、违章类型占比饼图、区域违章排行柱状图
- 支持按**日/周/月**切换时间维度，图表数据动态刷新
- **多条件组合查询**：依据车牌、驾驶员姓名、违章类型、时间区间检索违章记录

### 6. 系统管理模块

- 运维人员维护数据库基础配置、角色权限
- 配置随手拍奖励规则
- 管理外部摄像头 API 访问密钥
- 操作日志查看与管理

## 🛠️ 技术栈

### 前端
- **Vue 3** + **TypeScript** + Vite
- **Vue Router**（路由 + 权限守卫）+ **Pinia**（状态管理）
- **Element Plus**（UI 组件库）
- **Axios**（HTTP 请求封装）
- **ECharts**（数据可视化图表）
- 四端分离：市民端 / 审核工作台 / 统计分析台 / 系统管理端

### 后端
- **FastAPI** + Uvicorn（异步 Web 框架）
- **Pydantic**（数据校验）+ **SQLAlchemy**（ORM）
- **JWT** Authentication（登录鉴权）
- **Alembic**（数据库迁移管理）
- **PyMySQL**（MySQL 驱动）

### 数据层
- **MySQL 8.0** — 存储案件、车辆、违章、AI 结果、通知记录
- **Redis** — 异步任务队列 + 热点数据缓存
- **文件存储 / MinIO** — 保存原图、YOLO 标注图、车牌裁剪图

### AI 服务层
| 组件 | 功能 |
|------|------|
| **YOLOv8** | 检测车辆、车牌、红灯、停止线等视觉要素 |
| **OCR** | 识别车牌号码 |
| **规则判定服务** | 基于交通规则，综合检测结果生成疑似违章证据链 |
| **LLM / 多模态模型** | 生成 AI 初审意见；对疑难图片辅助复核；生成分析报告与整治方案 |

## 🏃 本地运行（数据库初始化）

> 后端在 `backend/` 下执行。运行中的 App 用 MySQL；测试用内存 SQLite，不需要 MySQL。

1. **建空库**（Alembic 只建表不建库）：
   ```sql
   CREATE DATABASE traffic_violation CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```
2. **装依赖**（项目用 uv，有 `uv.lock`）：
   ```bash
   cd backend
   uv sync --extra dev   # --extra dev 含 pytest/httpx，要跑测试就加
   uv pip install -r ../ai_service/requirements.txt  # YOLO、OpenCV、PaddleOCR
   ```
3. **配 `.env`**（gitignore，从示例拷贝后改账号密码）：
   ```bash
   cp .env.example .env
   # 把 DATABASE_URL 的账号密码改成自己本地 MySQL（默认 root:root@localhost:3306）
   ```
4. **跑迁移建表**：
   ```bash
   uv run alembic upgrade head
   ```
5. **灌演示数据**（幂等：建 4 个角色 + 默认管理员 `admin/admin1234` + 通知模板 + 演示车辆）：
   ```bash
   uv run python -m seed_data
   ```
6. **登录**：用 `admin / admin1234`。

### 启用邮件通知

注册验证码、找回密码验证码和违章通知共用 SMTP 配置。在 `backend/.env` 中填写：

```env
SMTP_HOST=smtp.example.com
SMTP_PORT=587
SMTP_USER=noreply@example.com
SMTP_PASSWORD=替换为 SMTP 授权码
SMTP_FROM=noreply@example.com
SMTP_SECURITY=starttls
SMTP_TIMEOUT_SECONDS=10
EMAIL_CODE_TTL_SECONDS=600
EMAIL_CODE_RESEND_SECONDS=60
EMAIL_CODE_MAX_ATTEMPTS=5
```

`SMTP_SECURITY` 支持 `starttls`、`ssl` 和 `none`。未配置 SMTP 时应用仍可启动，但邮件发送接口会记录失败；注册验证码接口返回 `503`，找回密码验证码接口为防止枚举账号始终返回 `202`。

### 启用 LLM 分析报告

统计分析页支持按日期范围汇总真实业务数据，并由 OpenAI-compatible 文本模型生成结构化中文报告。报告可在浏览器中打印或另存为 PDF，不会保存到数据库。

在 `backend/.env` 中配置：

```env
AI_PROVIDER=local
LLM_PROVIDER=zhipu
LLM_API_KEY=替换为实际密钥
LLM_BASE_URL=https://open.bigmodel.cn/api/paas/v4
LLM_TEXT_MODEL=glm-5.1
LLM_TIMEOUT_SECONDS=30
```

`LLM_PROVIDER` 也可填写 `openai_compatible` 并配套修改地址与模型。未配置真实 LLM 时，报告接口会返回服务不可用提示，不会生成占位报告。
